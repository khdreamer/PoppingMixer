Detailed Description

Reads a WAVE file from the app's resources or assets (iOS or Android respectively).

Note
On standard desktop systems (e.g. OSX and Windows) that do not use the concept of sandboxed apps this class is equivalent to the InputWaveFile class, i.e. the resourceFilenameWithExtension parameter is simply interpreted as a path (absolute or relative to the CWD otherwise).

Member Function Documentation

error_msg_t open ( char const *  resourceFilenameWithExtension)

Effect: Searches for resourceFilenameWithExtension in the app's resources and opens the file for reading.
Postconditions: Unless an error is reported, the file was successfully opened and its header parsed making it ready for audio and metadata extraction.

Effect: Reads interleaved numberOfSampleFrames * numberOfChannels() samples into pOutput and adjusts the current file position (as reported by remainingSamples()).
Preconditions:

  • a successful open() call
  • remainingSamples() >= samples
  • the buffer pointed to by pOutput must be large enough to hold numberOfSampleFrames * numberOfChannels() samples.
