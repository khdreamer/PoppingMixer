Quick start

Using the SpectrumWorx SDKs - quick start guide

Welcome to Using the SpectrumWorx SDK, a quick start guide. This guide aims to get the user up to speed quickly and to provide some introductory information on how to use the SDK. The focus of this tutorial is on differentiating between single-effect and multi-effect usage. Many effects comprise the SpectrumWorx SDK, they can be used individually as single, standalone effects but they can also be used as chained groups of effects. This tutorial describes the two use cases.

A. Standalone effect tutorial

Using a single effect is simpler and provides the user with a more deterministic output. For example, when using the Pitch Shifter, the user easily predicts that the output will be a pitch shifted signal. Or when using the Frecho, the user expects that this is an echo effect. However, when the two effects are chained and used together, it may be a bit harder to try and imagine what the output would sound like.

Signal flow

The signal flow in the case of single effect is simple:

Input Audio -> Effect -> Output Audio

For example:

Input Audio -> Pitch Shifter -> Output Audio
or
Input Audio -> Frecho -> Output Audio
or
Input Audio -> TuneWorx -> Output Audio
etc.

Standalone effect example

Let's see how this works in the code. Let's in this case use the Pitch Shifter. It should be a breeze to set-up. There is only one class that does it all: LE::Standalone::PitchShifter. Basic workflow consists of three steps:

  • Create a Standalone::PitchShifter instance.
  • Configure the instance.
  • Process that input data.

Step 1.

A Standalone::PitchShifter instance is created with the following code:

Standalone::PitchShifter::Ptr pPitchShifter( Standalone::PitchShifter::create() );

Step 2.

Once the standalone effect is created, it needs to be configured. Only the engine parameters are mandatory to be explicitly setup. User can just use the default values by calling:

pPitchShifter->setEngineParameters();

At this point, the effect-specific parameters should be set. Pitch Shifter has only two parameters:

pPitchShifter->set<PitchShifter::SemiTones>( 6 );
pPitchShifter->set<PitchShifter::Cents >( 50 );

This will shift the pitch of input audio up by 6.5 semitones.

Note
The user should note that there are 5 additional parameters that any SpectrumWorx effect has. Those are: start frequency, stop frequency, gain, wetness and bypass. These are automatically initialized with default values. If these defaults are not satisfactory, the parameters can be adjusted, like so:
pPitchShifter->set<SharedParameters::Gain>( 3 );
pPitchShifter->set<SharedParameters::Wet >( 70 );
pPitchShifter->setStartFrequencyInHz( 0 );
pPitchShifter->setStopFrequencyInHz ( 11025 );
This will process only the lower half of the frequency range, from 0 Hz to 11.025 Hz, the signal will be amplified by 3 decibel, and 70% of the processed signal will be mixed with 30% of the original signal.

Step 3.

The final step is data processing. Any number of input samples can be supplied to the process method.

while(mainChannelInputs)
pPitchShifter->process( mainChannelInputs, sideChannelInputs, outputs, numberOfSamples );
Note
In the case of the PitchShifter, sideChannelInputs is null. However, some effects like TalkingWind for example, need another audio channel (that we call Side) to achieve the right sound effect. And that's it for a single effect set-up!

B. Multi-effect tutorial

Chaining several effects together can give some very unexpected and unique sounds. Any number of individual effects can be chained which makes the sound possibilities limitless.

Signal flow

The signal flow looks like this:

Input Audio -> Effect1 -> Effect2 ->...->Effectx -> Output Audio

Here are some examples:

Input Audio -> Robotizer -> Frecho -> Output Audio
or
Input Audio -> Whisperer -> Reverser -> Pitch Spring -> Output Audio
or
Input Audio -> TuneWorx -> Slicer -> Output Audio
etc.

Multi-effect example

Let's see how this works in code. For this example we will use the PitchShifter and Freqverb effects, which will give us a pitch shifted-reverberant sound effect.
Input Audio -> Pitch Shifter -> Freqverb -> Output Audio
Using this in code is also as straightforward as it gets. It can be divided into two parts, the Engine and the Effects. The Engine part is embodied in the ModuleProcessor classes. Basic workflow consists of four steps:

  • Create and setup a ModuleProcessor instance.
  • Create and setup the desired effect(s).
  • Add the created effects to the ModuleProcessor's ModuleChain instance.
  • Process the input data.

Step 1.

A ChainedModuleProcessor is created with the following code:

ChainedModuleProcessor::Ptr pProcessor( ChainedModuleProcessor::create() );

Once the Module Processor is created, its parameters need to be initialized:

pProcessor->setEngineParameters();

Step 2.

The effects, in this example "pitch shifter" and "reverb" are created with the following lines:

Module<PitchShifter>::Ptr const pPitchShifter( createModule<PitchShifter>() );
Module<Freqverb >::Ptr const pFreqverb ( createModule<Freqverb >() );

After any effect has been created, its parameters need to be specified. PitchShifter's parameters are semitones and cents, in this case set to 3 semitones and 10 cents (3.1 semitones):

pPitchShifter->set<PitchShifter::SemiTones>( 3 );
pPitchShifter->set<PitchShifter::Cents >( 10 );

Freqverb's parameters as defined in the documentation are:

pFreqverb->set<Freqverb::Time60dB >( 5 );
pFreqverb->set<Freqverb::RoomSize >( -3 );
pFreqverb->set<Freqverb::ReverbPitch>( 0 );

Other effects are set up in a very similar way.

Step 3.

After the effects have been set-up, chaining needs to be done. In this example the pitch shifter and reverb effects are chained in the following order:

ModuleChain & moduleChain( pProcessor->moduleChain() );
moduleChain.append( pPitchShifter );
moduleChain.append( pFreqverb );

Step 4.

The final step is data processing. Any number of input samples can be supplied to the process method.

while(mainChannelInputs)
pProcessor->process( mainChannelInputs, sideChannelInputs, outputs, numberOfSamples );
Note
In the case of pitch shifter and reverb effects, sideChannelInputs is null. However, some effects like TalkingWind for example, need another audio channel (we call it Side) to achieve the right sound effect. And that’s it.
