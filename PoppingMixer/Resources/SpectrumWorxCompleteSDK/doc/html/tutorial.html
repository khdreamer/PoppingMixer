Tutorial

Sample app

A sample app for each supported platform is included in the SDK and is located in the /sample sub-folder. It demonstrates the API and general SpectrumWorx SDK usage.
The sample app has no GUI, it is a non-interactive command line app works in the following way:

  • read input audio from a file in the /sample/samples folder
  • send input audio through the SpectrumWorx engine and effects
  • write output audio to a file.
Note
The demo trial version of the SpectrumWorx SDK inserts random short silent periods into the output.

To build it use the provided IDE project:

Android

  • the SpectrumWorx SDK and the sample app are pure native C++ applications, no Java wrappers are provided
  • a pregenerated NDK project is located in the sample/Android folder
  • the Android environment requires additional setup, a brief outline of the required steps from a Windows command line is provided below:
    • cd into the folder.
    • "%ANDROID_SDK%\tools\android.bat" update project --path . --target android-<N>
      where <N> is the minimum required Android API level.
      Example:
      "c:\Program Files\Android\android-sdk\tools\android.bat" update project --path . --target android-10
    • for more information and/or troubleshooting consult the official Android NDK, ADT and/or Eclipse documentation
  • Eclipse: create empty workspace.
  • Eclipse: New Project -> Android -> Andorid Project from Existing Code
  • Eclipse: right-click-on-project -> Android Tools -> Add Native Support
  • Eclipse: right-click-on-project -> Debug As -> Android (Native) Application
  • the sample app itself (not the SpectrumWorx SDK) requires level 9 of the Android API and by default links with the development version of the ARMv7_NEON build

iOS & OSX

  • Xcode project located in the sample/Xcode folder
  • the SpectrumWorx SDK and the sample app use a purely C++ API, no Objective-C wrappers are provided

Windows

  • Visual Studio 2010 project located in the sample folder
  • the project can be safely upgraded and used with a later version of Visual Studio

The location of the generated output files can vary significantly between the different platforms: watch the output console/log window as the sample app will print the exact path for each generated output file. Embedded/mobile platforms may not allow direct filesystem access and may require additional steps to get to the generated files, consult the official documentation of the particular OS and/or hardware vendor for more information.

Sample code

The SpectrumWorx SDK is straightforward to use. It can be divided into two parts: the Engine and the Effects. The Engine part is embodied in the ModuleProcessor classes. As the example will show, a basic workflow consists of four steps. The user needs to:

  • create and setup a ModuleProcessor instance
  • create and setup the desired effect(s)
  • add the created effects to the ModuleProcessor's ModuleChain
  • process.

1.) A ChainedModuleProcessor is created with the following code:

ChainedModuleProcessor::Ptr pProcessor( ChainedModuleProcessor::create() );

Once the Module Processor is created, its parameters need to be initialized:

pProcessor->setEngineParameters();

2.) The effects, in this example "pitch shifter", "slew limiter", and "autotune" are created with the following lines:

Module<PitchShifter>::Ptr pPitchShifter( createModule<PitchShifter>() );
Module<SlewLimiter >::Ptr pSlewLimiter ( createModule<SlewLimiter >() );
Module<TuneWorx >::Ptr pTuneWorx ( createModule<TuneWorx >() );

After any effect has been created, its parameters need to be specified. All effects have 5 basic shared parameters: start frequency, stop frequency, gain, wetness and bypass. PitchShifter also has 2 specific parameters.

pPitchShifter->set<SharedParameters::Gain>( 3 );
pPitchShifter->set<SharedParameters::Wet >( 70 );
pPitchShifter->setStartFrequencyInHz( 0, *pProcessor );
pPitchShifter->setStopFrequencyInHz ( 11025, *pProcessor );

This will pitch-shift only the lower half of the frequency range (from 0 Hz to 11.025 Hz), the signal will be amplified by 3 dB and 70% of the pitch-shifted signal will be mixed with 30% of the original signal.

Pitch-shifter's own parameters are semitones and cents, in this case set to 6 semitones and 50 cents (6.5 semitones):

pPitchShifter->set<PitchShifter::SemiTones>( 6 );
pPitchShifter->set<PitchShifter::Cents >( 50 );

TuneWorx's (most important) own parameters are the target key and semitones. In this case we will "autotune" to key 'A', the first and fifth semitones:

pTuneWorx->set<TuneWorx::Key >( TuneWorx::Key::A );
pTuneWorx->set<TuneWorx::Semi01>( true );
pTuneWorx->set<TuneWorx::Semi05>( true );

Other effects are set up in a similar way.

3.) After the effects have been set-up, chaining needs to be done. In this example pitch-shifter, slew-limiter and autotune are chained in the following order:

ModuleChain & moduleChain( pProcessor->moduleChain() );
moduleChain.append( pitchShifter );
moduleChain.append( slewLimiter );
moduleChain.append( tuneWorx );

4.) The final step is data processing. Any number of input samples can be supplied to the process method.

pProcessor->process
(
mainChannelInputs,
sideChannelInputs,
outputs,
numberOfSamples
);
Note
Please explore the sources in the "sample/presets" subfolder for more information and more elaborate examples.
